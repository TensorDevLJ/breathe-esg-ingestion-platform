# Deployment Guide: Render.com

**Complete step-by-step guide to deploy Breathe ESG to production.**

---

## Prerequisites

- GitHub account with push access to repository
- Render.com account (free tier available)
- Django backend ready to deploy
- React frontend ready to build

---

## Step 1: Prepare Backend for Production

### 1.1 Set DEBUG = False

```python
# Backend/config/settings.py
DEBUG = False
```

### 1.2 Collect Static Files

```bash
python manage.py collectstatic --noinput
```

### 1.3 Create Render-specific Settings

```python
# Backend/config/settings.py (bottom of file)

# Render deployment
if not DEBUG:
    # Security
    SECURE_SSL_REDIRECT = True
    SESSION_COOKIE_SECURE = True
    CSRF_COOKIE_SECURE = True
    
    # Allow Render's domain
    ALLOWED_HOSTS = env.list('ALLOWED_HOSTS', default=[
        'breathe-esg-api.onrender.com',  # Your actual domain
    ])
```

### 1.4 Ensure requirements.txt is Updated

```bash
cd Backend
pip freeze > requirements.txt
git add requirements.txt
git commit -m "Update dependencies for production"
git push
```

## Step 2: Deploy Backend to Render

### 2.1 Create PostgreSQL Database

1. Log in to Render.com
2. Click **New +** → **PostgreSQL**
3. Fill in:
   - **Name**: `breathe-esg-db`
   - **Database**: `breathe_esg`
   - **User**: `postgres`
   - **Region**: Closest to your users
4. Click **Create Database**
5. **Copy the Internal Database URL** (you'll use this next)

### 2.2 Create Web Service for Backend

1. Click **New +** → **Web Service**
2. **Connect from GitHub**
   - Select your `breathe-esg` repository
   - Click **Connect**
3. **Configure**:
   - **Name**: `breathe-esg-api`
   - **Region**: Same as database
   - **Branch**: `main`
   - **Runtime**: `Python 3`
   - **Build Command**:
     ```bash
     pip install -r Backend/requirements.txt && \
     cd Backend && \
     python manage.py migrate && \
     python manage.py collectstatic --noinput
     ```
   - **Start Command**:
     ```bash
     cd Backend && \
     gunicorn config.wsgi:application --bind 0.0.0.0:$PORT
     ```

### 2.3 Set Environment Variables

In Render dashboard for `breathe-esg-api` service:

1. Go to **Environment** tab
2. Add these variables:

```
SECRET_KEY=<generate-random-key>
DEBUG=False
ALLOWED_HOSTS=breathe-esg-api.onrender.com
DATABASE_URL=<paste-from-PostgreSQL-service>
CORS_ALLOWED_ORIGINS=https://breathe-esg-ui.onrender.com,https://yourdomain.com

# Get from database service
DB_NAME=breathe_esg
DB_USER=postgres
DB_PASSWORD=<auto-generated>
DB_HOST=<internal-url>
DB_PORT=5432

# JWT
PYTHONUNBUFFERED=1
```

### 2.4 Monitor Deployment

1. Click **Logs** tab
2. Watch for deployment progress
3. Should see: `Application is up`

### 2.5 Verify Backend

Test API:
```bash
curl https://breathe-esg-api.onrender.com/api/health/
# Should return 200 OK
```

## Step 3: Deploy Frontend to Render (Static Site)

### 3.1 Build Frontend Locally

```bash
cd Frontend
npm install
npm run build
# Outputs to dist/
```

### 3.2 Create Static Site on Render

1. Click **New +** → **Static Site**
2. **Connect GitHub**
3. **Configure**:
   - **Name**: `breathe-esg-ui`
   - **Repository**: your repo
   - **Branch**: `main`
   - **Build Command**: `cd Frontend && npm install && npm run build`
   - **Publish Directory**: `Frontend/dist`

### 3.3 Set Environment Variables

In Render dashboard for `breathe-esg-ui`:

```
VITE_API_URL=https://breathe-esg-api.onrender.com/api
```

### 3.4 Monitor Deployment

Wait for build to complete (2-3 minutes)

Visit: `https://breathe-esg-ui.onrender.com`

## Step 6: Custom Domain (Optional)

### 6.1 Add Custom Domain to Frontend

1. Frontend service → **Settings** tab
2. **Custom Domain**:
   - Enter your domain: `esg.yourdomain.com`
   - Update DNS settings as instructed

### 6.2 Add Custom Domain to Backend

1. Backend service → **Settings** tab
2. **Custom Domain**:
   - Enter: `api.yourdomain.com`
   - Update DNS

### 6.3 Update CORS & Frontend URL

Backend environment variables:

```
CORS_ALLOWED_ORIGINS=https://esg.yourdomain.com
ALLOWED_HOSTS=api.yourdomain.com,breathe-esg-api.onrender.com
```

Frontend `.env`:

```
VITE_API_URL=https://api.yourdomain.com/api
```

1. Service → **Logs** tab

## Troubleshooting

### "Application failed to start"

**Check logs**:
```
Service → Logs → Look for error
```

**Common causes**:
- `ModuleNotFoundError`: Missing dependency in requirements.txt
- `DATABASE_URL not set`: Missing environment variable
- `SECRET_KEY not set`: Add to Render env vars

**Fix**:
1. Update `Backend/requirements.txt`
2. Set environment variable in Render
3. Redeploy

### "502 Bad Gateway"

**Cause**: Backend crashed or not responding

**Fix**:
1. Check backend logs
2. Look for Python errors
3. Restart service: Settings → **Restart**

### "CORS error in frontend"

**Error message**:
```
Access to XMLHttpRequest from origin has been blocked by CORS policy
```

**Fix**:
1. Backend → Environment variables
2. Update `CORS_ALLOWED_ORIGINS` to include frontend URL
3. Redeploy backend

### "Database connection refused"

**Cause**: DATABASE_URL incorrect or database not ready

**Fix**:
1. Check PostgreSQL service is **Available**
2. Copy **Internal Database URL** from PostgreSQL service
3. Paste into Backend `DATABASE_URL`
4. Redeploy backend

## Rollback Procedure

If deployment breaks:

1. **Get last working commit**:
   ```bash
   git log --oneline
   ```

2. **Revert to that commit**:
   ```bash
   git revert <commit-hash>
   git push
   ```

3. **Render auto-redeploys** on push

4. **Verify deployment** in Logs
